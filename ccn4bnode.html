CCN4B.js

CCN4B.js - a Javascript framework for CCNx Protocol.
The CCN4B.js project is an open source project which is a proof of concept. Short for Content Centric Networking for Browsers, CCN4B is a small CCNx client API written in Javascript (JS), that utilizes the evented model of JS development. APIs are written to be asynchronous. Web application developers can choose to deploy this API in the client side of the web application (in script tag), or on the server-side of the web application, as a module import for a Server-Side Javascript platform called Node.js.

About CCN4B.js Brief history, browser support, and more

What is CCN4B.js

The CCN4B.js project is a collection of Javascript libraries and services that support communication and development using the CCNx Protocol. It constitutes a mix of Client-Side Javascript Libraries and Server-Side Middleware.


Brief History

This project was started in August of 2011 in response to an effort to help solve a problem with getting broadband and data services deployed in the developing world. CCNx offers an interesting set of attributes which may support various use cases for deployment in adverse conditions, such as intermittent power, poor quality of internet connectivity, lack of connectivity, and lack of general Internet infrastructure.

Browser As the Platform

Since 2007, with the advent of the iPhone/iPod Ecosystem, the importance of the Web Runtime has grown in leaps and bounds. With the rise and fall of proprietary development platforms, and with the continued issues with IPR around technologies like Android, the Web is the only platform that is future proof and cannot be erase, held under injunction, sued, or turned off.

Low Cost/Embedded Devices to Bridge The Digital Diviide

The BOM cost of a complete Wifi enabled device with a screen is on a downward curve, with power and capability increasing as cost declines over time. This is a challenge for profits of large semiconductor companies, but presents a great opportunity for underdeveloped countries to leap ahead in technology deployment and utilization. The Web is the lowest common denominator for app development and deployment on these low cost devices (after SMS).

Usage - coding, APIs, samples, and deployment.

Quickstart Guide

  1. Install node.js - (nodejs.org)
  2. Install NPM (npmjs.org)
  3. run 'npm install ccn4bnode'. This will install ccnx-0.4.0
  4. do a git clone of ccn4bnode 'git clone git@github.com:truedat101/ccn4bnode.git'
  5. cd ccn4bnode/jumbosocket
  6. run 'node ccn4bnode-server.js'
  7. go to http://localhost:8080/ccn4bnode

CCN4B.js source code

  • CCN4B.js - umbrella project hosted on Github.com, under BSD license (currently a placeholder project on Github)
  • CCN4BName.js - handles CCNx Naming and URI validation and encoding, hosted on Github, under BSD license

CCN4B Motivation & Usecases

Objective: Create a Clusters of Micro-Clouds: little nodes that can aggregate content between nodes and client devices for the developing world

  • Broadband may be widely unavailable
  • Mobile internet is not a universal right
  • WiFi is not always on
  • Cost of (mobile and fixed-lin) broadband services may rise, not fall, over time, if competition in a particular country decreases
  • 24/7 always-on power is a luxury
  • Traditional Consumers are an anomaly unique to developed countries
  • Governments may drive innovation more than capitalist market forces
  • Content and Apps must be free, as the traditional ways to monetize are not used outside of formal economies
  • Traditional Intellectual Property rights are hard to uphold or enforce

Why Javascript (JS)?

  • JS is one of the most commonly understood languages
  • JS is increasingly important as the glue between the major pieces of the web runtime: HTML5/CSS3/DOM
  • JS is based on open standards, and tracks to an open specification in active development
  • JS-based VMs/runtimes have major financial support from Mozilla, Apple, and Google.
  • JS performance has increased dramatically with optimizations and improvements in VM technology
  • Front-end JS developers fetch a premium in the market
  • Server-Side development of JS as a web middleware is being rapidly adopted by developers formerly doing PHP, Java, or ruby

Implementation

Various Approaches

  • Browser Plugin to manage mapping of URLs to CCNx names, and services
  • Pure client-side library for creating CCNx packets and communicating with CCNx services
  • Web middleware proxy, developed as an Apache Mod or similar
  • Node.js Middleware as a proxy to CCNx services, and a CCNx client API to communicate with CCNx services

Status

  • Early Sept 2011 - Publish first CCNx package into NPM, called ccn4bnode
  • Early Setp 2011 - Present CCN4B.js at CCNxCon 2011 @ PARC
